## [Проект №1: Оптимизация процесса клиентской поддержки в банке «Сберегай»](https://github.com/yanakushner/Practicum_projects/tree/main/Проект%201)
Вертикаль: FinTech / Banking

### Решаемая проблема: 
Низкий уровень удовлетворенности клиентов  из-за длительного времени ожидания ответа (до 2-х суток по почте) и перегрузки телефонной линии (80% времени операторов). Отсутствие оперативного сбора обратной связи и высокая доля типовых запросов, обрабатываемых вручную.
Команда: Индивидуальный проект

### Что сделано:
- Проведено интервьюирование стейкхолдеров  и аудит текущих процессов для выявления «болей» и точек роста.
- Спроектирован процесс AS-IS в нотации BPMN 
- Разработана карта пользовательских историй (User Story Map)
- Спроектирован целевой процесс TO-BE 
- Созданы Lo-Fi прототипы интерфейсов мобильного приложения 
- Подготовлена и проведена презентация решения для бизнеса 

### Результат: 
Спроектировано комплексное решение для повышения удовлетворенности клиентов до 8/10. Автоматизация типовых запросов позволила снизить нагрузку на первую линию поддержки и сократить время ожидания ответа с нескольких суток до режима реального времени.
### Стек: 
BPMN 2.0, User Story Mapping, Miro, Wireframing, Интервьюирование стейкхолдеров.

## [Проект №2: Система автоматизации логистики и учёта инвентаря «Л-Логистика» для сети отелей ](https://github.com/yanakushner/Practicum_projects/tree/main/Проект%202)
Вертикаль: HoReCa / Supply Chain

### Решаемая проблема: 
Неэффективное ручное планирование логистики чистого белья между 7 отелями сети и центральной прачечной. Отсутствие синхронизации между загрузкой отелей и поставками, что приводило к дефициту белья, высоким операционным расходам и непрозрачному учету остатков.
Команда: Индивидуальный проект 

### Что сделано:
- Выявлены недостатки процессов AS-IS  и сформирован «Отчёт об обследовании»  и «Требования к решению» 
- Разработаны алгоритмы расчета потребности в белье на основе интеграции через API с системой управления отелем 
-Разработаны UML диаграммы (Классов, объектов, вариантов использования) 
- Разработано техническое задание (ТЗ) 
- Подготовлена программа и методика испытаний (ПМИ) 

### Результат: 
Спроектирована единая экосистема для управления движением ТМЦ. Решение позволило автоматизировать формирование графиков доставки, оптимизировать маршруты водителей и обеспечить прозрачный контроль инвентаризации во всей сети.
### Стек: 
UML (Use Case, Class, Object Diagrams), API Integration, BPMN, ПМИ, Отчет об обследовании, ТЗ, Waterfall/Cascade.

## [Проект №3: Реинжиниринг системы онлайн-продаж для гипермаркета стройматериалов «Переделкино»](https://github.com/yanakushner/Practicum_projects/tree/main/Проект%203)
Вертикаль: E-commerce / Retail

### Решаемая проблема: 
Высокая доля ручного труда (администраторы вручную подтверждают наличие каждой позиции по телефону), из-за чего подтверждение заказа занимало до 24 часов. Отсутствие синхронизации остатков сайта с учетной системой склада, примитивный UX (нет поиска и фильтров, ошибки в корзине) и нестабильная работа онлайн-оплаты, что приводило к потере клиентов.

### Что сделано:
- Проведен полный аудит бизнес-процессов и составлен Отчет об обследовании 
- Составлена карта пути клиента (CJM)  
- Спроектированы процессы AS-IS и TO-BE в нотации BPMN 
- Разработан комплексный документ требований (SRS) 
- Описаны детальные сценарии взаимодействия в формате Use Cases
- Спроектирован прототип будущего сайта магазина 
- Разработаны UML-диаграммы классов и объектов для моделирования структуры каталога, заказов и личного кабинета 
- Описана логика API-интеграции между сайтом, складской программой учета и платежным шлюзом
- Разработана пользовательская инструкция 
- Подготовлена видеопрезентация решения 

### Результат:
Разработан проект полной трансформации онлайн-магазина. Внедрение предложенных решений позволило автоматизировать проверку остатков, сократить время обработки заказа до нескольких минут, внедрить систему лояльности и обеспечить стабильный прием платежей.
### Стек: 
BPMN, UML (Class, Object, Use Case), CJM, User Story Mapping, API Design, Figma/Miro (Prototyping), Отчет об обследовании, Требования к решению.
